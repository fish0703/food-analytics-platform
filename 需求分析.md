# 餐饮点评大数据可视化系统 —— 需求分析文档

## 1. 项目背景
本系统为课程设计项目，旨在通过对大众点评、美团网等平台的餐饮数据进行采集与分析，利用可视化技术直观展示城市餐饮消费特征、品类分布、消费与评价的关系以及趋势变化，最终构建一个简易的可视化网站。

## 2. 数据需求
### 2.1 数据来源
- 通过爬虫抓取大众点评/美团网的商家信息，或采用公开数据集（如Kaggle、天池等）。
- 数据应至少包含以下字段：
  - 商家名称
  - 所属城市
  - 美食类型（一级分类，如火锅、川菜、日料等）
  - 人均消费（数值，单位：元）
  - 综合评价指数（评分，如1~5分）
  - 口味指数（评分）
  - 环境指数（评分）
  - 评价数量（数值）
  - 经纬度（可选，用于地图展示）

### 2.2 数据规模
- 覆盖不少于3个城市，每个城市不少于500条有效商家记录。

## 3. 功能需求
### 3.1 数据收集（后台处理，无需前端展示）
- 完成爬虫或数据集导入，清洗后存入本地文件（CSV/JSON）或数据库。

### 3.2 地图分布展示
- 在地图上标注各城市商家分布，使用气泡图或散点图，气泡大小表示商家数量，颜色表示平均人均消费或总点评数。
- 提供城市级别的统计表格：城市名称、商家数量、点评数量、平均人均消费。

### 3.3 统计分析（共7项，需全部可视化呈现）
| 编号 | 分析内容 | 可视化方式 |
|------|----------|------------|
| 1 | 城市商家数量与美食类型的关系 | 堆叠柱状图（城市为X轴，各类型堆叠）或热力图（城市×类型，颜色深浅表示数量） |
| 2 | 不同城市最钟爱的美食类型（Top5） | 分组柱状图或并列饼图（按城市筛选） |
| 3 | 人均消费与评价指数的关系 | 散点图（X=人均消费，Y=评分），可添加趋势线 |
| 4 | 人均消费与综合指数的关系 | 同上（综合指数即综合评分） |
| 5 | 口味指数与环境指数的关系 | 散点图（X=口味分，Y=环境分），用颜色区分人均消费档位 |
| 6 | 评价数量与评价指数的关系 | 散点图（X=评价数量，Y=评分），观察异常点 |
| 7 | 趋势变化（如啤酒衰退、重庆小面崛起） | 若数据含时间字段则用折线图展示品类占比变化；若无时间字段，则用品类占比对比图（当前各城市啤酒类商家占比 vs 重庆小面占比）或人均消费档位分布间接反映 |

### 3.4 可视化网站展示
- 将所有图表整合至单一HTML页面或简易仪表盘（可使用ECharts、Plotly、Streamlit等工具）。
- 页面布局：
  - 顶部：KPI数字卡片（总商家数、总点评数、覆盖城市数、平均人均消费）
  - 主体：依次排列上述7张图表 + 地图 + 城市统计表格
  - 交互（可选）：下拉菜单筛选城市、美食类型，图表联动更新。

## 4. 非功能需求
- 页面响应式，能在主流浏览器（Chrome/Firefox/Edge）正常显示。
- 图表加载流畅，数据量≤10万条时无明显卡顿。
- 代码注释完整，便于课程答辩讲解。
